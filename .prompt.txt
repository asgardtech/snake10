The Game component defines:

When `pauseGame()` is called, the Game component sets `gameState = 'paused'`, but `GameEngine.pause()` sets its state to `'idle'`. This means the two state machines become temporarily out of sync:
- Game thinks: `paused`
- Engine thinks: `idle`

While this works functionally (because the game loop checks `gameState === 'playing'`), it's a design smell that could cause confusion and bugs in future refactoring.

**Recommendation:** Either document this design decision with a comment, or update `GameEngine` to support the 'paused' state explicitly for better semantic alignment.

The old code was:
```typescript
const { stop } = useGameLoop({ ... });
```

The new code is:
